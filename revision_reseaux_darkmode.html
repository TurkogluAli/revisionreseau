Web & HTTP

HTTP = protocole sans état (stateless), utilise TCP. Port 80 (HTTP), 443 (HTTPS = HTTP + TLS).
VersionConnexionParticularité
HTTP/1.0Non persistante1 objet par connexion TCP — 2 RTT par objet
HTTP/1.1Persistante (défaut)Plusieurs objets par TCP, pipelining possible
HTTP/2Persistante + TLSMultiplexage (plusieurs flux sur 1 connexion)
HTTP/3QUIC (sur UDP)0-RTT si serveur connu, élimine head-of-line blocking
Calcul RTT : Non-persistant = 2RTT + ttrans par objet. Persistant avec pipelining ≈ 2RTT initiaux + 1RTT pour tous les objets.

Établissement & fermeture TCP

3-way handshake (ouverture) :
1. Client → Serveur : SYN, seq=x
2. Serveur → Client : SYN-ACK, seq=y, ack=x+1
3. Client → Serveur : ACK, ack=y+1

Fermeture : 4-way — FIN → ACK → FIN → ACK

Protocole IP

IP = best effort, sans connexion, sans garantie. Chaque paquet routé indépendamment. RFC 791.

Adressage IPv4 & CIDR

Format : 32 bits — 192.168.1.0/24 → masque 255.255.255.0

Calcul : /n → 2^(32-n) adresses totales, dont 2^(32-n) - 2 utilisables
Adresse réseau = IP AND masque | Broadcast = réseau OR NOT(masque)
